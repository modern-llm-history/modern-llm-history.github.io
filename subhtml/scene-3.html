2022-11 – 2024-08

ChatGPT Moment

2022年11月30日、OpenAI は ChatGPT を公開しました。s3-01

sec-1 と sec-2 で見てきたように、技術の芯そのものはすでに存在していました。GPT-3 はありましたし、InstructGPT もありました。では何が変わったのか。答えはかなり単純で、入口の形が変わったのです。

それまで巨大言語モデルは、研究者が論文で語るものか、開発者が API を通じて自分の製品に埋め込むものでした。一般の人にとっては、せいぜい「なんだかすごいらしい」という噂話に近かった。でも ChatGPT は違いました。ブラウザを開けば、そこに会話相手がいる。登録すれば、すぐ話しかけられる。しかも返事は、従来のチャットボットよりずっと自然でした。s3-01

「巨大言語モデルという技術が存在する」と、「自分がいまここでそれと会話できる」は、似ているようでまったく違います。前者はニュースで、後者は体験です。ニュースは見出しで終わることがありますが、体験は人に話したくなる。2022年末から2023年初頭にかけて、まさにそれが起きました。学生がレポートの下書きを作らせ、プログラマーがバグ修正を相談し、SNS には「これ、未来じゃないか」という反応があふれました。

ChatGPT は、巨大言語モデルを研究者の話題から、一般社会の出来事へと引きずり出したのです。

3-1. ChatGPT の衝撃: なぜ「InstructGPT の姉妹」が社会現象になったのか

先述のとおり、OpenAI は ChatGPT を「InstructGPT の sibling model（姉妹モデル）」と説明しました。s3-01
この表現はとても重要です。というのも、ChatGPT はゼロから突然降ってきた新発明ではなかったんですね。sec-2 で見た InstructGPT は、すでに「指示に従う」「人間が好む答え方をする」という方向へ大きく踏み出していました。ChatGPT は、その流れを一般公開向けの会話体験にまで仕上げたものだったのです。

では、なぜそれほど爆発したのか。

まず大きかったのは、UI と UX です。
UI は user interface、つまり人が機械に触れる窓口のこと。UX は user experience、つまり使っていてどんな体験になるかのことです。OpenAI は ChatGPT に、メールソフトやメッセンジャーのような、ごく直感的な会話画面を与えました。プロンプトだとか few-shot だとかを知らなくても、とりあえず「質問を書いて送る」だけで動く。これは技術の普及では繰り返し起きることですが、中身の性能と同じくらい、あるいはそれ以上に重要なんです。

次に、返事の自然さ。
従来の対話ボットは、少し長く話すとすぐに破綻したり、妙にテンプレートっぽかったり、会話が噛み合わなかったりしました。ChatGPT にも誤答はたくさんありましたが、それでも、最初の数往復については「たしかに会話している感じ」がかなり強かった。人間は短い会話の局面では相手に一貫した人格や理解を読み込みやすいので、その体験はとても強く感じられます。sec-1 で見た AI Dungeon と似ていますね。あれも、局所的な「それっぽさ」が人を驚かせた。ただし ChatGPT では、その「それっぽさ」の持続時間と安定性がはるかに上がっていたのです。

そして決定的だったのは、無料で、誰でも試せた こと。
巨大言語モデルが大衆に届くには、論文でも API でも足りません。人々が自分の手で使い、「これで何ができるか」を自分の生活の中で発見できなければならない。ChatGPT はその条件を満たしていました。

成長の速さは象徴的でした。公開から5日で100万ユーザーを集め、2023年1月には月間アクティブユーザーが1億人に達したと UBS の推計が報じられます。Reuters はこれを、当時としては消費者向けアプリ史上もっとも速い立ち上がりのひとつとして伝えました。s3-02

ここで月間アクティブユーザーという言葉も押さえておきましょう。
英語では MAU（Monthly Active Users）。「その月に少なくとも一度はそのサービスを使った人の数」です。毎日使っている人と月に一回だけ触った人を同じ一人として数える粗い指標ではありますが、「どれだけ広く届いたか」を見るには便利です。ChatGPT の場合、この数字の伸びが「これは一部の好事家だけの玩具ではなく、社会的な現象だ」と各社に悟らせました。

sec-1 で AI Dungeon が「巨大言語モデルを体験できる最初期の入口」だったとすれば、ChatGPT は桁違いに大きな入口でした。AI Dungeon が教えてくれたのは「AI と遊ぶと未来っぽい」でしたが、ChatGPT が教えたのは「AI は自分の仕事に使えるかもしれない」です。この差は、のちの経済的風景をまったく変えることになります。

3-2. GPT-4

ChatGPT が入口を変えたのに対して、2023年3月の GPT-4 は天井を引き上げました。s3-03

GPT-4 は、GPT-3.5 世代と比べて、長い指示への追従、複雑な文章理解、プログラミング、数学、専門試験のような問題で大きく性能を伸ばしました。加えて、画像を入力として読めるようになった。つまりテキストだけでなく画像も理解対象に含めるマルチモーダル対応が話題になりました。s3-03

マルチモーダルという言葉は難しそうですが、意味は単純です。
モーダルとは「情報の種類」くらいの意味で、テキスト、画像、音声、動画などを指します。テキストしか扱えないモデルは単一モーダル、テキストと画像の両方を扱えるならマルチモーダル。それだけのことです。

GPT-4 が特別に見えた理由は、単に点数が高かったからではありません。これまでできなかったことが、かなりの精度でできる領域が一気に増えたからです。GPT-4 Technical Report では、各種の試験ベンチマークや専門テストで大幅な改善が報告されました。s3-03 もちろんベンチマークは万能ではありません。けれどこの時期、少なくとも公開情報ベースでは、「GPT-4 はそれ以前のモデル群より一段上」という認識がかなり広く共有されました。

このあたりから、何をもって「すごい」と言うのかも変わっていきます。

sec-1 で見た GLUE は、自然言語処理の研究室的な総合模試でした。
それに対して、この時期に共通言語になっていくのは、MMLU、GSM8K、Chatbot Arena のような指標です。

MMLU（Massive Multitask Language Understanding）は、大学レベルを含む幅広い知識問題を大量に集めたテスト。s3-39
GSM8K は算数文章題を集めたベンチマークで、「知識を知っているか」ではなく「途中の計算や推論をきちんと積み上げられるか」を見やすい指標としてよく使われました。s3-40
Chatbot Arena は、二つのモデルの返答を匿名で人間に見せ、どちらが好ましいかを投票してもらう対戦型の評価です。s3-23

ここには、評価軸のかなりはっきりした変化が見えます。
GLUE が「言語を理解しているか」を問うていたのに対し、この時期の指標は「考えているように見えるか」「会話として役に立つか」「人間がどちらを好むか」を問う。sec-2 の InstructGPT で始まった「人間の好みに合わせる」という流れが、評価の枠組みそのものにまで波及してきたわけです。

コラム: 「マルチモーダル」って何？ AI が文字以外を扱うとはどういうことか

「GPT-4 は画像も読めます」「GPT-4o は音声も自然です」「Gemini は長い動画も理解できます」
こう言われると、つい「AI が人間みたいに目や耳を持った」と感じてしまいます。でも、実際にはそういう魔法が起きているわけではありません。

まず押さえたいのは、巨大言語モデルの基本が依然としてトークン列の処理だということです。
トークンというのは、単語そのものというより、「文章を細かく刻んだ部品」くらいのもの。テキストモデルは、この部品の並びを受け取り、次に来そうな部品を予測します。

では画像はどうやって扱うのか。大きく二つの考え方があります。

ひとつは、外付けの視覚エンコーダーを使うやり方。
画像をまず別のネットワークで読み取り、その結果を言語モデルが扱いやすい形に変換してから、本体の言語モデルへ渡す方式です。初期の代表例としてよく挙げられるのが 2022年の Flamingo です。s3-41
たとえるなら、画像を見て要点だけをメモしてくれる通訳がいて、そのメモを言語モデルが読む感じですね。

もうひとつは、画像や音声も最初からトークン化して、できるだけ同じ枠組みで扱う方向。
テキストも画像も音声も、最終的には「並んだ部品列」として処理しようとします。実装はもちろん簡単ではありませんが、この方向だと、異なる種類の情報をより自然にまたいで扱いやすくなります。

なぜこれが大事なのか。
理由は、人間の仕事がテキストだけで完結していないからです。メールもあれば、PDF もあり、表計算もあり、図表もあり、画面操作もあります。AI が本当に「助手」に近づくなら、文字だけでなく、図・音・画面・ファイルをまたいで扱えたほうが有利ですよね。

ただし、宣伝と実力は分けて考える必要があります。
「画像入力に対応」と言っても、細かい図の読み違いは普通に起きますし、「長い動画を理解」と言っても、要点の取りこぼしや、見ていない部分を見たように話すこともあります。マルチモーダル化は確かに前進ですが、「人間のように理解した」と即断してはいけない。むしろ、こうした機能が増えるほど、何ができて何ができないのかを測るのは難しくなります（これについては sec-4 でも触れます）。

3-3. 競争の幕開け

GPT-4 のあと、各社が本格的な競争に突入します。
ここは少し登場人物が増えますが、大事なのは個別の名前よりも、それまで数社の先行技術だったものが、一気に産業全体の競争領域になった という構図のほうです。

Google は対応が早かったものの、この時期は試行錯誤も目立ちました。
2023年2月に Bard という会話 AI サービスを発表しましたが、当初の中身は LaMDA という対話特化モデルの系譜にありました。その後 2023年12月に Gemini という新しいモデル系列が発表され、2024年2月にはサービス名も Gemini へ統一されます。名前が変わり続けること自体が、この時期の各社の試行錯誤の激しさを物語っています。s3-04 s3-05 s3-06

Anthropic は OpenAI の出身者たちが設立した会社で、「役に立つ・無害・正直」という調整方針を掲げていました。2023年3月に Claude を公開し、長文読解や文体の安定感で評価されます。2023年7月の Claude 2、2024年3月の Claude 3 系列へと進む中で、最前線の一角を占めるようになりました。s3-07 s3-08 s3-09

そして、ここが大切なのですが、この競争は閉じたモデルだけで起きたわけではありません。

Meta は 2023年2月に LLaMA を研究者向けに発表しました。s3-10 LLaMA は Meta の大規模言語モデル系列で、のちに Llama 2、Llama 3 へと続きます。s3-11 s3-12 重要なのは、LLaMA 系列がモデルの重みを外部に公開する流れの中心にあったことです。この話はすぐ後の 3-4 で詳しく見ます。

さらに 2023年後半にはフランスの Mistral AI が Mistral 7B を公開しました。7B は70億パラメータの意味で、GPT-4 のような巨大モデルに比べれば小さい。しかし軽量で扱いやすく、Apache 2.0 ライセンスで利用しやすかったことから、企業や個人が自分向けに調整するベースとして広く使われました。日本でも楽天グループが 2024年に Rakuten AI 7B を発表するなど、この流れは各国に波及していきます。s3-13 s3-14

2024年に入ると、競争はさらに枝分かれします。
Google は 2024年2月に Gemini 1.5 Pro を発表し、100万トークンという非常に長いコンテキスト長を打ち出しました。s3-15
OpenAI は 2024年5月、GPT-4o を公開し、テキスト・画像・音声をより統合した対話へ進みます。s3-16

ここでいうコンテキスト長とは、一度にモデルが読める情報量の上限のことです。
短い会話しか覚えていられない相手と、長い資料を丸ごと読み込める相手では、できる仕事が違いますよね。コンテキスト長が長いほど、長文の文書、長い会話履歴、大きなコードの一部などを一度に扱いやすくなる。ただし、「長く読める」ことと「長い内容を本当にうまく使いこなせる」ことは別なので、この時期の各社の宣伝文句は、そのまま能力の保証ではない点にも注意が必要でした。

こうして振り返ると、2023年から2024年前半にかけて起きたことの本質は、技術の進歩そのものよりも、参加者の爆発的な増加でした。もはや「OpenAI が先行している技術」ではなく、Google、Anthropic、Meta、Mistral、各国の研究者と企業が参入する巨大な市場になっていたのです。

3-4. オープンウェイトの潮流: 「自分で持ちたい」という欲望

ChatGPT と GPT-4 の物語は、どうしても OpenAI や Anthropic のような閉じたモデル提供者の話になりがちです。
けれど同時に、別の流れも育っていました。モデルの重み、つまり、覚えた中身を公開し、外の人々が自分で動かし、改造し、細かく調整する流れです。

その中心にあったのが、Meta が公開した LLaMA 系列でした。さらに llama.cpp のような軽量な実行環境が急速に発達したことで、巨大モデルを個人の PC でも動かせる可能性がぐっと広がりました。sec-1 で AI Dungeon の運営コストが「1日1万ドル以上」だった話を覚えているでしょうか。あの時代には非現実的だった「自分でモデルを動かす」が、ここで初めて現実的な選択肢に入り始めたのです。s3-10 s3-19 s3-20

なぜ「自分で持つ」ことがそれほど大事なのか。
API 経由でモデルを使う世界では、利用者はあくまで「借り手」です。価格も仕様も停止条件も、基本的には提供者が決めます。対してオープンウェイトの世界では、モデルを自社サーバーに置ける。機密データを外へ出さずに使える。自分たちの用語やルールに合わせて微調整できる。sec-1 で見た ファインチューニングを、LoRA や QLoRA といったより軽量な方法で、個人や中小企業でも試せるようになりました。s3-22

ただし、「小さいモデルをうまく微調整すれば、大きなモデルにだいたい追いつける」という当初の期待は、大部分が幻想でした。特定のベンチマークではよく見えても、現実の仕事の乱雑さの前では、巨大な汎用モデルの総合力に及ばないことが多かったのです。s3-23

それでもこの流れが重要だったのは、巨大言語モデルを「一部の巨大企業の専有物」ではなく、「エコシステムそのもの」へ変えたからです。モデル本体だけでなく、推論エンジン、量子化ツール、データセット、微調整手法、評価系、操作画面まで含めた巨大な周辺産業が、この時期に育ち始めました。

コラム: オープンウェイトの世界をもう少し詳しく

AI の議論では、「オープン」という言葉がしばしば雑に使われます。
でも実際には、何が開いているのかで意味が大きく変わるんです。

まず open-source（オープンソース）。
本来は、プログラムのソースコードが公開され、ライセンスに従って利用・改変・再配布できる状態を指します。AI で言えば、学習や推論に使うコード、更には学習に使ったデータセットまでが公開されている、というのがまず第一歩。

次に open-weight（オープンウェイト）。
モデルの重み、つまり学習済みパラメータが利用者に渡されることです。ソースコードが公開されていても重みがなければ同じモデルは動かせませんし、逆に重みだけあっても学習データや学習過程が非公開なら、完全再現はできません。最近の「オープンモデル」と呼ばれるものの多くは、厳密には open-source ではなく open-weight に近いです。

最後に open-access（オープンアクセス）。
もっと広く、「外部からアクセスできる」程度の意味で使われることがあります。研究者だけに申請制で重みを配る場合も、API 経由で試せるだけの場合も、文脈しだいでは open-access と言えてしまう。つまり、この言葉だけでは自由度がわかりません。

なぜこの区別が重要か。
「公開されているから安全」「公開されているから自由」「公開されているから再現できる」とは限らないからです。

  • コードだけ公開でも、同じモデルは再現できないかもしれない
  • 重みがあっても、商用利用に厳しい制限があるかもしれない
  • API で試せても、内部の学習データも重みも見えないかもしれない

Llama 2 や Mistral 7B が広く使われたのは、単に性能だけでなく、「ある程度自由に動かせる」という実務上の意味があったから。逆に GPT-4 や Claude は非常に強力でしたが、基本的には借り物です。
この違いは、単なる思想の違いではありません。コスト構造、データの取り扱い、内製の自由度、事業の差別化に直結する、かなり現実的な違いなのです。

ChatGPT 以前にも GPT-NeoX や BLOOM のような大規模な公開系モデルは存在していました。s3-17 s3-18
しかし当時は、個人がローカルで巨大モデルを動かすための道具立てがまだ未成熟でした。重みがあっても、動かすのが重い。量子化も、軽量ランタイムも、簡単な操作画面も、十分には整っていなかったんです。

流れが変わったのは、LLaMA でした。
Meta は LLaMA を研究者向けに配布しましたが、ほどなくして重みが非公式に広く出回ります。s3-10 s3-20 そしてその前後で、llama.cpp のような軽量推論ランタイムが急速に発達しました。llama.cpp は、比較的限られた計算資源でも LLaMA 系モデルを動かしやすくするソフトウェアで、量子化と相性がよく、個人 PC やノート PC での実行可能性を一気に押し上げました。s3-19

ここで役立ったのが、LoRA や QLoRA といった軽量なファインチューニング手法です。
LoRA は、モデル全部を再学習し直すのではなく、ごく小さな追加パラメータだけを学ばせることで、低コストに調整する考え方。QLoRA はその延長で、量子化したモデルに対しても実用的な微調整を可能にしました。s3-22

これによって、「自社の FAQ に詳しいモデル」「自社の文体に寄せたモデル」「この業務だけは得意なモデル」を比較的低いコストで作れるかもしれないという期待が生まれました。しかし実際には、小さなモデルを狭いタスク向けに調整しただけでは、巨大な汎用モデルの総合力に及ばないことが多かった。しかも評価方法しだいで「追いついたように見える」ことさえある。MT-Bench や Chatbot Arena を提案した研究自体が、評価方法の設計が結果をかなり左右しうると指摘しています。s3-23

3-5. ツール呼び出し: 「文章を書く機械」から「道具を選ぶ機械」へ

ChatGPT 以後の巨大言語モデルについて考えるとき、もうひとつ忘れてはいけない転換があります。
モデルが単に文章を生成するだけでなく、外部の道具を使う判断をするようになったことです。

たとえばユーザーが「今日の東京の天気を教えて」と聞いたとします。
言語モデルが訓練時の記憶だけで答えようとすると、最新の天気はわかりません。ここで必要なのは、天気 API を呼び出し、その結果を受け取って自然な文に整形すること。

この一連の流れを、人間が全部プログラムで書いてもよいのですが、ある時点から、「どの道具を使うべきか」自体をモデルに選ばせる設計が広まるんですね。

発想の前史として重要なのが ReAct（2022年）です。モデルに「考える」と「行動する」を交互に行わせる枠組みを示しました。s3-25
その後 2023年の Toolformer は、モデルが必要な場面で API 的な呼び出しを自分で挿入できることを示しました。s3-24
そして 2023年6月、OpenAI が function calling を API の正式機能として導入したことで、この発想は一気に実際のサービスへ流れ込みます。s3-26

function calling をやさしく言い換えるなら、
「モデルが、自分で全部答える代わりに、必要なら外部サービスへ注文を出せるようにする仕組み」です。

ただし、モデルがいきなりプログラムを自由自在に操れるようになったわけではありません。
実際には、開発者があらかじめ「こんな道具がありますよ」と一覧を与え、入出力の形を決めておきます。モデルはその中から選んで、決められた形式で呼び出す。かなり枠のある自由です。でも、それでも意味は大きかった。

なぜなら、これによって巨大言語モデルは閉じた知識の箱である必要がなくなったからです。
最新情報は検索すればよい。社内資料はデータベースから引けばよい。計算は Python に任せればよい。つまり、「全部を内部に覚えていなくても、外部とつながることで役に立つ」方向が本格化した。

この頃によく使われるようになったのが RAG（Retrieval-Augmented Generation）です。s3-27
RAG は、質問に関連する文書を外部から検索してコンテキストに詰め込み、そのうえでモデルに回答させる手法です。日常語で言えば、本を何冊か机の上に積んでから答えさせるようなもの。モデル単体にうろ覚えで答えさせるのではなく、「参考資料はこれです」と渡してから考えさせる。これだけで出力の質が大きく改善することがあります。

ただし、この時期のツール利用や RAG は、まだ sec-4 で見る本格的なエージェントとは距離があります。
多くの場合は、「一回質問を受けたら、一回か二回ツールを呼ぶ」程度でした。長い計画を立て、失敗してやり直し、何十分も自律行動する、という段階ではまだありません。

それでも、sec-2 で「次の単語を予測するだけの機械」だった巨大言語モデルが、ここでは「適切な補助輪を付ければ外の世界に働きかけられる機械」に変わりつつあった。この技術的な土台は、のちのエージェント時代の基本部品になっていきます。

コラム: ツール呼び出しは実際にはどう動いているのか

「モデルがツールを呼ぶ」と言うと、つい何か魔法のように聞こえます。
でも実際の仕組みは、かなり地味なんです。

まず開発者は、モデルにこう伝えます。

  • こんなツールがあります
  • ツール名はこれです
  • どういう引数を受け取ります
  • どういう形式で返ってきます

たとえば weather というツールがあり、city という引数を取る、と決めておくわけです。

そのうえでユーザーが「東京の天気は？」と聞くと、モデルは普通の日本語を返す代わりに、たとえば

{"tool": "weather", "city": "Tokyo"}

のような構造化された出力を返します。
アプリ側はそれを見て、本当に天気 API を叩き、結果を受け取ります。最後に、その結果を再びモデルへ渡して、「この結果を自然な日本語にして返して」とやる。利用者には会話が一続きに見えますが、裏では

  1. ユーザー発話
  2. モデルがツール選択
  3. アプリが実際にツール実行
  4. モデルが結果を文章化

という往復が走っているわけです。

ここで大事なのは、モデルがツールそのものを動かしているわけではない こと。
実際に API を叩いたり、データベースへ問い合わせたり、ファイルを読んだりするのは外側のプログラムです。モデルは「どの道具を使うべきか」を決め、その注文票を書く役割に近い。

ではなぜそれが便利なのか。
自然言語が非常に柔らかいインターフェースだからです。人間は「来週の会議を木曜午後にずらして、関係者にも共有して」と曖昧に頼めますよね。モデルはそれを「これはカレンダー操作だ」「参加者へメールも必要だ」と分解し、適切なツールへつなげられるかもしれない。つまり、自然言語が操作系の入口になっていくんです。

もちろん、誤ったツール選択や変な引数生成は普通に起こります。
だから function calling は「完全自律化」の完成ではなく、「自然言語で道具箱にアクセスするための中間層」ができた、と理解するのが近いでしょう。そしてこの中間層が、後のエージェント時代の基本部品になります。

3-6. この時期の経済的風景: 資本が雪崩れ込み、社会が「使い道」を学び始めた

sec-1 では、Microsoft が OpenAI に10億ドルを出資した話をしました。当時それは「わかる人だけがわかる」出来事でした。
sec-3 では、桁が変わります。

2023年1月、Microsoft は OpenAI への「複数年・数十億ドル規模」の追加投資を発表し、主要報道はその規模を約100億ドルと報じました。s3-28 もはや「大型スタートアップへの出資」ではなく、「国家規模のインフラへの賭け」に近い金額です。

Anthropic にも巨額資本が流れ込みます。Amazon が 2023年9月に最大40億ドルの投資を発表し、Google も最大20億ドル規模の投資を進めていると報じられました。s3-29 s3-30
フランスの Mistral も、設立直後の 2023年6月に約1.05億ユーロを調達し、年末にはさらに4.15億ドルの大型調達を行っています。s3-31 s3-32 設立から半年で5億ドル以上というのは、この時期の熱量を凝縮した数字です。

これらの数字が示しているのは、単に AI 企業が人気だということではありません。
sec-2 で見たように、巨大言語モデルは計算資源をどれだけ確保できるかが競争力の核心にある産業です。データセンター、GPU、クラウド、電力。優秀な研究者だけいても足りない。その意味で、ChatGPT 以後の AI ブームは、ソフトウェアの流行であると同時に、重たいインフラ産業でもありました。

その象徴が NVIDIA です。
NVIDIA は GPU を製造する企業で、もともとはゲーム向けグラフィックスで知られていましたが、深層学習の時代に「AI を作るための計算機」を売る企業として圧倒的な存在になりました。2024年2月に発表された同社の決算は、データセンター需要の急増をはっきり映し出しています。s3-33 生成 AI ブームが「金脈」だとすれば、NVIDIA はよく「ツルハシを売る会社」と呼ばれました。ゴールドラッシュで一番確実に儲かったのは、金を掘る人ではなく道具を売った人だった、という有名な話になぞらえているわけです。

一方、企業への導入も少しずつ見えてきます。

Palantir は 2023年4月に AIP を発表し、既存の業務へ LLM を組み込む方向を打ち出しました。s3-34
Salesforce は 2023年3月に Einstein GPT を発表し、顧客管理や営業支援ソフトへ生成 AI を統合しようとしました。s3-35
Notion も 2023年2月に Notion AI の一般提供を開始します。s3-36

面白いのは、この時期の企業導入が「仕事を全部置き換える」より、既存の仕事の一部を少し速くする方向だったことです。
会議メモの要約、メール文案、ドキュメントの叩き台、コード補完、FAQ 応答。どれも限定的ですが、限定的だからこそ導入しやすい。ここに sec-2 から続く一貫性があります。sec-2 で GitHub Copilot が「コードの次の一行を提案する」ことで価値を出したように、不完全でも、補助なら十分に価値になるのです。

ただし 2023年には、奇妙な停滞感もありました。

GPT-4 が作った壁がかなり高かったのです。
少なくとも Chatbot Arena の人間評価では、2024年3月に Claude 3 Opus が初めて明確に GPT-4 を上回ったと受け止められるまで、GPT-4 はかなり長いあいだ基準点であり続けました。s3-09 s3-38

この「一気に社会へ広まったのに、能力の最前線は少し落ち着いて見えた」時間は重要でした。社会が使い道を学ぶ猶予になったからです。

企業はこの時期、巨大言語モデルの限界もはっきり学びました。
幻覚、つまりもっともらしい誤答。長い文脈での取り違え。専門業務における信用性不足。検証ループのない高リスク用途への不向き。2024年の Stanford HAI AI Index も、生成 AI の導入が急増する一方で、信頼性やガバナンスの課題が引き続き大きいことをまとめています。s3-37

だから 2022年11月から2024年夏までのボトルネックをひとことで言うなら、信頼性と論理思考能力でした。

普及という意味でのハードルは、ChatGPT がかなり壊してしまった。けれど、「本当に任せてよいか」「長い仕事を最後までやりきれるか」という意味では、まだ弱かった。コーディングのように検証しやすい分野では早く価値が出ましたが、逆に言えば検証しづらい分野では、まだ「すごい補助ツール」の域を出ていなかったのです。

そして sec-4 で見るのは、その次の変化です。
モデルは、答える前により長く考えるようになり、ツールをまたいでより長く動くようになり、評価そのものが追いつかなくなっていきます。巨大言語モデルは、補助ツールから、限定的ではあっても「ある仕事を代行するもの」に近づき始めます。
